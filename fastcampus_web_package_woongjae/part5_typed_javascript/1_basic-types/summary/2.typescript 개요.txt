TypeScript Feature?
1. 타입 스크립트는 자바스크립트를 확장한 스크립트이다.
2. 자바스크립트를 안다면, 타입스크립트는 시간을 절약해주고 오류를 잡아주며, 코드 실행 전에
이를 고쳐준다.
3. 어떠한 자바스크립트 실행 환경, 브라우저에서 실행 할 수 있다.
4. 오픈소스 이다.

TypeScript?
- 타입스크립트는 프로그래밍 언어이다.
- 타입스크립트는 컴파일 언어이다. (전통적인 컴파일 언어: C++, Java와는 조금 다름)
- Compile 대신 Transpile이라는 용어를 사용하기도 함.
- 인터프리트 언어이다.(자바스크립트와 같은)
- 타입스크립트는 컴파일 시 이해 할 수 있는 자바스크립트 언어로 해석해준다.

Compiled vs Interpreted
- Compiled?
  * 컴파일이 필요
  * 컴파일러가 필요
  * 컴파일하는 시점이 존재(컴파일 타임)
  * 컴파일 된 결과물을 실행
  * 컴파일 된 결과물을 실행하는 시점이 존재(런타임)

- Interpreted?
  * 컴파일이 필요 없음
  * 컴파일러가 필요 없음
  * 컴파일 하는 시점이 없음
  * 코드 자체를 실행
  * 코드를 실행하는 시점이 존재(런타임)


- TypeScript execute?
타입 스크립트를 작성하면 이를 브라우저나, node.js 환경의 런타임에서 바로 사용이 불가함.
따라서 TypeScript Compiler에 의해 컴파일(Transpile타임)에 변환해주어야 함.
이 역할을 타입스크립트 컴파일러가 해주며, 이를 plain javascript로 변환!!!
